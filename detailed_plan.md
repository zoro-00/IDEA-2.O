These are non-negotiable features that form the backbone of a credible AML platform demo.

|---|---------|---------------|-------------------|-----------------|-------------------|-------------------|

| C4 | **Entity Risk Scoring Engine** | Every entity (account, person, business) needs a composite risk score. # 3. Features to Skip for MVP

| # | Feature | Why Skip It | What To Say If Asked |
|---|---------|------------|---------------------|
| S1 | Full KYC/CDD integration | Requires external data sources (Dow Jones, World-Check), API costs, complex entity resolution | "Our architecture supports pluggable KYC data sources — in production, we'd integrate World-Check and Dow Jones risk feeds via our entity enrichment pipeline." |
| S2 | Full LSTM/GRU temporal sequence models | Training time, data requirements, and tuning effort are prohibitive for a hackathon. Simpler temporal aggregations + EWMA achieve 80% of the demo value. | "Our temporal analysis layer uses EWMA and windowed statistical methods. The architecture supports LSTM sequence models for production-scale temporal pattern mining." |
| S3 | Autoencoder-based anomaly detection | Isolation Forest gives you 90% of the demo value in 10% of the implementation effort. Autoencoders need careful architecture tuning. | "We use Isolation Forest for unsupervised anomaly detection. Our ML pipeline is designed to swap in deep autoencoder models as the system matures." |
| S4 | Full Temporal Graph Networks (TGN/TGAT) | Cutting-edge research, extremely hard to implement correctly, and impossible to debug in a hackathon | "Our GraphSAGE model operates on snapshot graphs. Our architecture roadmap includes TGAT for continuous-time dynamic graph learning." |
| S5 | Full regulatory reporting (CTR, STR formats) | XML/XBRL formatting for FinCEN BSA E-Filing is purely bureaucratic and adds zero demo value | "SAR narratives are auto-generated. Production integration with FinCEN E-Filing and local regulatory formats is a configuration exercise." |
| S6 | Role-based access control (RBAC) | Security is critical in production but invisible in a demo. Time spent on auth is time not spent on features. | "Production deployment includes role-based access aligned to the three lines of defense model." |
| S7 | Full audit trail/logging | Same as RBAC — essential in production, invisible in demo | "Every action is logged to an immutable audit trail for regulatory examination readiness." |
| S8 | Node2Vec embeddings | GraphSAGE subsumes much of Node2Vec's value and is more impressive to show. Node2Vec is a preprocessing step that's hard to demo standalone. | "Our GNN architecture learns node embeddings jointly with the classification task, similar to but more powerful than Node2Vec." |
| S9 | Fuzzy name matching / entity resolution | Complex NLP problem that requires dedicated infrastructure (dedupe libraries, phonetic algorithms). Fake it with exact matching on synthetic data. | "Our entity resolution pipeline supports fuzzy matching with Jaro-Winkler, phonetic encoding, and BERT-based semantic similarity." |
| S10 | Multi-tenancy / white-labeling | Product feature, not a hackathon feature | "STAR is designed as a multi-tenant SaaS platform with institution-level data isolation." |

### Architecture Decisions Rationale

| Decision | Rationale |
|----------|-----------|
| **Neo4j as sole database** | Simplifies stack massively. Neo4j handles graph + property storage. For hackathon, no need for PostgreSQL alongside it. Alerts, cases, risk scores all stored as graph nodes/relationships. |
| **In-memory caching over Redis** | Python dictionaries + FastAPI state are sufficient. No need to add Redis to the stack. |
| **No Kafka/message queue** | Asyncio producer → detection engine → WebSocket is sufficient for demo scale. Mention Kafka in architecture slides. |
| **LLM via API, not local** | GPT-4/Claude API is far more reliable and capable than running a local LLM. Budget $20 for API calls. |
| **PyTorch Geometric only if needed** | GraphSAGE is the only GNN worth implementing. If time is short, Isolation Forest + graph analytics alone are sufficient. |

### Composite Risk Score Formula

```python
# risk/composite_scorer.py

class CompositeRiskScorer:
    """
    Enterprise-grade composite risk scoring engine.
    
    Combines multiple risk signals into a single 0-100 score with
    full decomposition for regulatory explainability.
    
    Weights are calibrated based on AML domain expertise:
    - Rule-based signals are high-confidence, legally defined → 30%
    - Graph topology is uniquely powerful for network detection → 30%  
    - Behavioral anomaly catches novel patterns → 20%
    - ML models add probabilistic intelligence → 20%
    """
    
    # Weight configuration — tunable per deployment
    WEIGHTS = {
        'rule_risk':        0.30,   # Threshold + velocity + structuring rules
        'graph_risk':       0.30,   # PageRank + community + cycle involvement
        'behavioral_risk':  0.20,   # Isolation Forest + EWMA deviation
        'ml_risk':          0.20,   # XGBoost + GraphSAGE
    }
    
    def calculate(self, entity_id: str) -> RiskBreakdown:
        # ── 1. RULE-BASED RISK (0-100) ──────────────────────────
        rule_score = self._calculate_rule_risk(entity_id)
        
        # ── 2. GRAPH TOPOLOGY RISK (0-100) ──────────────────────
        graph_score = self._calculate_graph_risk(entity_id)
        
        # ── 3. BEHAVIORAL ANOMALY RISK (0-100) ──────────────────
        behavioral_score = self._calculate_behavioral_risk(entity_id)
        
        # ── 4. ML MODEL RISK (0-100) ────────────────────────────
        ml_score = self._calculate_ml_risk(entity_id)
        
        # ── 5. COMPOSITE CALCULATION ────────────────────────────
        composite = (
            self.WEIGHTS['rule_risk'] * rule_score +
            self.WEIGHTS['graph_risk'] * graph_score +
            self.WEIGHTS['behavioral_risk'] * behavioral_score +
            self.WEIGHTS['ml_risk'] * ml_score
        )
        
        # ── 6. BOOST FACTORS (multiplicative) ───────────────────
        # These represent high-confidence signals that should
        # amplify the overall score
        boost = 1.0
        
        if self._involved_in_cycle(entity_id):
            boost *= 1.3   # Cycle involvement is extremely suspicious
            
        if self._is_dormant_reactivated(entity_id):
            boost *= 1.2   # Dormant reactivation amplifies other signals
            
        if self._high_risk_jurisdiction(entity_id):
            boost *= 1.15  # Jurisdictional risk amplifier
        
        # Apply boost, cap at 100
        final_score = min(100.0, composite * boost)
        
        # ── 7. RISK LEVEL CLASSIFICATION ────────────────────────
        risk_level = self._classify_risk_level(final_score)
        
        return RiskBreakdown(
            entity_id=entity_id,
            composite_score=round(final_score, 1),
            risk_level=risk_level,
            components={
                'rule_risk': {
                    'score': round(rule_score, 1),
                    'weight': self.WEIGHTS['rule_risk'],
                    'weighted_contribution': round(self.WEIGHTS['rule_risk'] * rule_score, 1),
                    'details': self._rule_details
                },
                'graph_risk': {
                    'score': round(graph_score, 1),
                    'weight': self.WEIGHTS['graph_risk'],
                    'weighted_contribution': round(self.WEIGHTS['graph_risk'] * graph_score, 1),
                    'details': self._graph_details
                },
                'behavioral_risk': {
                    'score': round(behavioral_score, 1),
                    'weight': self.WEIGHTS['behavioral_risk'],
                    'weighted_contribution': round(self.WEIGHTS['behavioral_risk'] * behavioral_score, 1),
                    'details': self._behavioral_details
                },
                'ml_risk': {
                    'score': round(ml_score, 1),
                    'weight': self.WEIGHTS['ml_risk'],
                    'weighted_contribution': round(self.WEIGHTS['ml_risk'] * ml_score, 1),
                    'details': self._ml_details
                }
            },
            boost_factors=self._boost_details,
            boost_multiplier=round(boost, 2)
        )
```

### Sub-Score Calculations

```python
    def _calculate_rule_risk(self, entity_id: str) -> float:
        """
        Rule-based risk: 0-100
        
        Scoring logic:
        - Each triggered rule contributes a fixed score
        - Multiple triggers are additive (capped at 100)
        - More severe rules contribute more
        """
        score = 0.0
        details = []
        
        # Check structuring
        structuring = self.structuring_detector.check(entity_id)
        if structuring['detected']:
            severity = min(40, structuring['txn_count'] * 8)  # Up to 40 pts
            score += severity
            details.append(f"Structuring: {structuring['txn_count']} txns "
                         f"below $10K threshold (${structuring['total']:,.0f} total)")
        
        # Check velocity
        velocity = self.velocity_detector.check(entity_id)
        if velocity['detected']:
            severity = min(30, velocity['deviation_sigma'] * 10)  # Up to 30 pts
            score += severity
            details.append(f"Velocity anomaly: {velocity['deviation_sigma']:.1f}σ "
                         f"above baseline")
        
        # Check threshold (large transactions)
        threshold = self.threshold_detector.check(entity_id)
        if threshold['detected']:
            severity = min(20, threshold['count'] * 5)  # Up to 20 pts
            score += severity
            details.append(f"Large transactions: {threshold['count']} txns "
                         f"above ${threshold['threshold']:,.0f}")
        
        # Check dormancy reactivation
        dormancy = self.dormancy_detector.check(entity_id)
        if dormancy['detected']:
            score += 25
            details.append(f"Dormant reactivation: {dormancy['dormant_days']} days "
                         f"dormant, then {dormancy['recent_txns']} recent txns")
        
        self._rule_details = details
        return min(100.0, score)
    
    
    def _calculate_graph_risk(self, entity_id: str) -> float:
        """
        Graph topology risk: 0-100
        
        Combines:
        - PageRank percentile (high centrality = high risk)
        - Community risk (avg risk of community members)
        - Cycle involvement (binary boost)
        - Connected component analysis
        """
        score = 0.0
        details = []
        
        # PageRank percentile — normalized to 0-100
        # Top 1% PageRank → ~40 points
        pagerank = self.neo4j.get_property(entity_id, 'pagerank')
        pagerank_percentile = self.neo4j.get_percentile(entity_id, 'pagerank')
        pr_score = min(40, pagerank_percentile * 0.4)
        score += pr_score
        details.append(f"PageRank: {pagerank:.4f} "
                      f"(top {100-pagerank_percentile:.0f}% centrality)")
        
        # Community risk — if most community members are risky, this entity is too
        community_id = self.neo4j.get_property(entity_id, 'community_id')
        community_avg_risk = self.neo4j.get_community_avg_risk(community_id)
        community_score = min(25, community_avg_risk * 0.25)
        score += community_score
        details.append(f"Community #{community_id}: avg risk {community_avg_risk:.1f}")
        
        # Degree analysis — unusually high degree suggests hub/mule
        degree = self.neo4j.get_property(entity_id, 'degree')
        degree_percentile = self.neo4j.get_percentile(entity_id, 'degree')
        if degree_percentile > 90:
            degree_score = min(20, (degree_percentile - 90) * 2)
            score += degree_score
            details.append(f"High degree: {degree} connections "
                         f"(top {100-degree_percentile:.0f}%)")
        
        # Cycle involvement — binary, adds 15 points
        cycles = self.cycle_detector.find_cycles_for_entity(entity_id)
        if cycles:
            score += 15
            details.append(f"Involved in {len(cycles)} transaction cycle(s)")
        
        self._graph_details = details
        return min(100.0, score)
    
    
    def _calculate_behavioral_risk(self, entity_id: str) -> float:
        """
        Behavioral anomaly risk: 0-100
        
        Combines:
        - EWMA deviation (how far from historical baseline)
        - Isolation Forest anomaly score
        - Z-score on key features
        """
        score = 0.0
        details = []
        
        # EWMA deviation on transaction amount
        ewma_result = self.ewma_detector.check(entity_id, metric='txn_amount')
        if ewma_result['deviation_sigma'] > 2:
            ewma_score = min(40, ewma_result['deviation_sigma'] * 10)
            score += ewma_score
            details.append(f"Amount EWMA: {ewma_result['deviation_sigma']:.1f}σ "
                         f"above baseline")
        
        # EWMA deviation on transaction frequency
        ewma_freq = self.ewma_detector.check(entity_id, metric='txn_frequency')
        if ewma_freq['deviation_sigma'] > 2:
            freq_score = min(30, ewma_freq['deviation_sigma'] * 8)
            score += freq_score
            details.append(f"Frequency EWMA: {ewma_freq['deviation_sigma']:.1f}σ "
                         f"above baseline")
        
        # Isolation Forest anomaly score (sklearn returns -1 for outliers)
        features = self.feature_extractor.extract(entity_id)
        if_score = self.isolation_forest.score_samples([features])[0]
        # Convert: more negative = more anomalous
        # Typical range: -0.5 (normal) to -0.7+ (anomalous)
        if if_score < -0.55:
            anomaly_score = min(30, (abs(if_score) - 0.55) * 200)
            score += anomaly_score
            details.append(f"Isolation Forest: anomaly score {if_score:.3f}")
        
        self._behavioral_details = details
        return min(100.0, score)
    
    
    def _calculate_ml_risk(self, entity_id: str) -> float:
        """
        ML model risk: 0-100
        
        Combines:
        - GraphSAGE node classification probability
        - XGBoost (optional) tabular classification probability
        
        If GraphSAGE is not available, falls back to boosted
        Isolation Forest score
        """
        score = 0.0
        details = []
        
        # GraphSAGE suspicious probability (0-1)
        gnn_score = self.neo4j.get_property(entity_id, 'gnn_score')
        if gnn_score is not None:
            gnn_contribution = gnn_score * 60  # Up to 60 points
            score += gnn_contribution
            details.append(f"GraphSAGE: {gnn_score:.2f} suspicious probability")
        
        # XGBoost classification (if available)
        features = self.feature_extractor.extract(entity_id)
        if self.xgboost_model:
            xgb_prob = self.xgboost_model.predict_proba([features])[0][1]
            xgb_contribution = xgb_prob * 40  # Up to 40 points
            score += xgb_contribution
            details.append(f"XGBoost: {xgb_prob:.2f} suspicious probability")
        else:
            # Fallback: use Isolation Forest as second ML signal
            # Already computed in behavioral — just add extra weight
            if_score = self.isolation_forest.score_samples([features])[0]
            if if_score < -0.55:
                fallback_score = min(40, (abs(if_score) - 0.55) * 250)
                score += fallback_score
                details.append(f"Isolation Forest (ML fallback): {if_score:.3f}")
        
        self._ml_details = details
        return min(100.0, score)
    
    
    def _classify_risk_level(self, score: float) -> str:
        """Risk level thresholds calibrated for AML operations."""
        if score >= 80:
            return "critical"      # Immediate investigation required
        elif score >= 60:
            return "high"          # Priority investigation within 24 hours
        elif score >= 40:
            return "medium"        # Review within 5 business days
        else:
            return "low"           # Monitor, no immediate action
```
